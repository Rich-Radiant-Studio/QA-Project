# 🚀 批量翻译策略

由于 APP 有 27 个文件需要翻译，且 HomeScreen.js 等文件非常大（包含大量数据），我采用以下高效策略：

## 📋 翻译策略

### 1. 核心 UI 文本优先
只翻译用户看到的界面文本：
- 按钮文本
- 标签文本  
- Placeholder
- Alert 消息
- 提示文本

### 2. 数据内容保持原样
不翻译数据内容：
- 城市名称（纽约、伦敦等）
- 用户名（张三、李四等）
- 示例问题标题
- 话题名称

**原因：** 这些是示例数据，实际使用时会从服务器获取，不需要硬编码翻译。

### 3. 批量处理
我会一次性翻译多个文件，提高效率。

考虑到效率，我建议：
