## 使用流程

### 1. 登录 Expo 账号（首次）

```bash
eas login
```

如果没有账号，会提示注册。

### 2. 构建包含更新功能的 APK

**重要**: 必须重新构建一次 APK，让热更新功能生效。

```powershell
# 方式 A: 本地构建（推荐）
.\scripts\build-android.ps1

# 方式 B: 使用 EAS 构建
eas build --platform android --profile production
```

### 3. 发布更新

修改代码后，发布更新：

```bash
# 发布到生产环境
eas update --branch production --message "修复了登录问题"

# 发布到预览环境
eas update --branch preview --message "测试新功能"
```

### 4. 用户获取更新

用户下次打开 APP 时：
1. APP 自动检查更新
2. 后台下载新版本
3. 重启 APP 后生效

## 更新配置说明

### checkAutomatically

控制何时检查更新：

- `ON_LOAD`: 每次启动时检查（推荐）
- `ON_ERROR_RECOVERY`: 仅在错误恢复时检查
- `NEVER`: 从不自动检查

### fallbackToCacheTimeout

下载更新的超时时间（毫秒）：

- `0`: 立即使用缓存版本，后台下载更新
- `30000`: 等待30秒下载更新，超时则使用缓存

如果发现问题，可以回滚到之前的版本：

```bash

# 回滚到指定版本
eas update:republish --group <group-id>
```

## 监控更新

### 查看更新统计

```bash
eas update:view <update-id>
```

### 在代码中监控更新

```javascript
import * as Updates from 'expo-updates';

// 检查更新
const checkForUpdates = async () => {
  try {
    const update = await Updates.checkForUpdateAsync();
    if (update.isAvailable) {
      await Updates.fetchUpdateAsync();
      // 提示用户重启
      Alert.alert(
        '更新可用',
        '发现新版本，是否立即更新？',
        [
          { text: '稍后', style: 'cancel' },
          { text: '立即更新', onPress: () => Updates.reloadAsync() }
        ]
      );
    }
  } catch (error) {
    console.error('检查更新失败:', error);
  }
};
```

## 常见问题

### Q1: 更新后用户需要重启 APP 吗？

A: 是的，更新会在后台下载，用户下次重启 APP 时生效。

### Q2: 可以强制用户立即更新吗？

A: 可以，在代码中检测到更新后调用 `Updates.reloadAsync()`。

### Q3: 更新失败怎么办？

A: Expo Updates 有自动回滚机制，如果新版本崩溃，会自动回滚到上一个稳定版本。

### Q4: 可以针对特定用户发布更新吗？

A: 可以，使用不同的分支（branch）或渠道（channel）。

### Q5: 更新包有多大？

A: 通常只有几百 KB 到几 MB，取决于修改的内容。
