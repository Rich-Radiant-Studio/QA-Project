# 分类选择弹窗优化说明

## 优化目标

解决分类选择弹窗空间不足的问题，优化用户体验。

## 优化方案

采用两个视图切换的方式，而不是在一个页面同时显示一级和二级分类。

### 视图1：一级分类列表
- 显示所有一级分类的大卡片
- 每个卡片显示图标、名称、描述和右箭头
- 点击任意一级分类，切换到视图2

### 视图2：二级分类列表
- 只显示选中的一级分类下的所有二级分类
- 标题显示一级分类的名称
- 提供返回按钮，可以返回视图1
- 点击二级分类后，确认选择并关闭弹窗

## 实现细节

### 新增状态
```javascript
const [categoryModalView, setCategoryModalView] = useState('level1'); // 'level1' 或 'level2'
const [tempSelectedLevel1, setTempSelectedLevel1] = useState(null); // 临时选中的一级分类
```

### 选择逻辑

#### 选择一级分类
```javascript
const selectLevel1 = (cat) => {
  setTempSelectedLevel1(cat);
  setCategoryModalView('level2'); // 切换到二级分类视图
};
```

#### 选择二级分类
```javascript
const selectLevel2 = (cat) => {
  setSelectedLevel1(tempSelectedLevel1); // 确认选择一级分类
  setSelectedLevel2(cat); // 选择二级分类
  setShowCategoryModal(false); // 关闭弹窗
  setCategoryModalView('level1'); // 重置视图
  setTempSelectedLevel1(null); // 清空临时选择
};
```

#### 返回一级分类
```javascript
const backToLevel1 = () => {
  setCategoryModalView('level1');
  setTempSelectedLevel1(null);
};
```

### UI变化

#### 标题栏
- 视图1：显示"选择问题类别" + 关闭按钮
- 视图2：显示一级分类名称 + 返回按钮

#### 内容区域
- 视图1：一级分类列表（大卡片，带描述）
- 视图2：二级分类列表（小标签，网格布局）

#### 一级分类卡片
- 移除了选中状态的高亮和勾选图标
- 右侧显示右箭头，表示可以进入下一级
- 点击后直接切换视图，不需要二次确认

## 优点

1. **空间利用更好**
   - 每个视图只显示一种类型的分类
   - 避免了滚动查看的问题

2. **用户体验更好**
   - 操作流程清晰：选一级 → 选二级 → 完成
   - 视图切换是零延迟的（数据已在本地）
   - 可以随时返回重新选择

3. **视觉更清晰**
   - 一级分类用大卡片展示，更醒目
   - 二级分类用小标签展示，更紧凑
   - 减少了视觉干扰

4. **性能优化**
   - 只渲染当前视图需要的内容
   - 减少了DOM节点数量
   - 提升了渲染性能

## 用户操作流程

1. 点击"问题类别"选择器
2. 弹窗显示所有一级分类
3. 点击某个一级分类（如"国家"）
4. 弹窗切换显示该分类下的所有二级分类
5. 点击某个二级分类（如"政策法规"）
6. 选择完成，弹窗关闭，显示"国家 > 政策法规"

如果想重新选择：
- 在二级分类视图点击返回按钮
- 返回一级分类列表
- 重新选择

## 技术实现

- 使用状态控制视图切换，无需额外的动画库
- 数据结构不变，只改变展示方式
- 保持了原有的数据加载和错误处理逻辑
- 兼容原有的本地备用数据降级方案
