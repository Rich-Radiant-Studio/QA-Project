## 环境判断逻辑

### 开发环境 (Development)

**触发条件**:
- 使用 `npm start` 启动
- 使用 `npm run android` 运行
- `__DEV__` 为 `true`

**特点**:
- ✅ 显示详细日志
- ✅ 启用热重载
- ✅ 显示开发者菜单
- ✅ 使用 debug.keystore 签名

### 生产环境 (Production)

**触发条件**:
- 使用 `.\scripts\build-android.ps1` 构建
- 使用 `gradlew assembleRelease` 构建
- `__DEV__` 为 `false`

**特点**:
- ✅ 代码压缩和混淆
- ✅ 资源优化
- ✅ 使用 release.keystore 签名
- ✅ 禁用开发者工具
- ✅ 性能优化

## 开发工作流

### 日常开发

```bash
# 启动开发服务器
npm start

# 或直接运行 Android
npm run android
```

**环境**: 开发环境  
**特点**: 热重载、详细日志、快速调试

### 测试 Release 版本

```powershell
# 构建 Release APK
.\scripts\build-android.ps1

# 安装到设备测试
adb install -r android\app\build\outputs\apk\release\app-release.apk
```

**环境**: 生产环境  
**特点**: 真实的生产环境表现

### 发布到应用商店

```powershell
# 构建最终版本
.\scripts\build-android.ps1

# 验证签名
& "$env:ANDROID_HOME\build-tools\35.0.0\apksigner.bat" verify --print-certs android\app\build\outputs\apk\release\app-release.apk

# 上传到应用商店
# android\app\build\outputs\apk\release\app-release.apk
```

## 常见场景

### 场景 1: 需要使用 Mock 数据

编辑 `src/config/env.js`:

```javascript
const USE_MOCK = true;  // 改为 true
```

### 场景 2: 部分接口用 Mock，部分用真实

编辑 `src/config/env.js`:

```javascript
const USE_MOCK = false;  // 全局使用真实服务器

const API_MODE_CONFIG = {
  '/questions': 'mock',        // 这个接口用 Mock
  '/app/user/auth/login': 'real',  // 这个接口用真实服务器
};
```

### 场景 3: 切换到不同的服务器

编辑 `src/config/env.js`:

```javascript
const ENV = {
  dev: {
    apiUrl: 'http://新的服务器地址',
    contentApiUrl: 'http://新的服务器地址',
  },
};
```

## 环境变量优先级

1. **USE_MOCK = true** → Mock 环境（最高优先级）
2. **SIMULATE_PRODUCTION = true** → 生产环境配置（已关闭）
3. **__DEV__ = true** → 开发环境
4. **__DEV__ = false** → 生产环境（Release 构建）

### 方法 3: 检查构建类型

```powershell
# 查看 APK 签名
& "$env:ANDROID_HOME\build-tools\35.0.0\apksigner.bat" verify --print-certs android\app\build\outputs\apk\release\app-release.apk
```

- Debug 版本: `CN=, OU=, O=, L=, ST=, C=`（空白）
- Release 版本: `CN=QA Hero, OU=Mobile Development, ...`（有内容）
