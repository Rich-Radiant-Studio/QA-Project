# 热更新策略配置

## 当前配置：提示用户更新

✅ 已配置为**提示用户更新**模式

### 用户体验流程

1. 用户打开 APP
2. APP 检查是否有更新
3. **弹出提示框**：
   ```
   发现新版本
   我们发布了新版本，包含功能改进和问题修复。
   是否立即更新？
   
   [稍后]  [立即更新]
   ```
4. 用户选择：
   - **立即更新**：下载并重启 APP
   - **稍后**：后台下载，下次启动时生效

## 两种策略对比

### 策略 1：静默自动更新

**配置**：
```json
{
  "updates": {
    "checkAutomatically": "ON_LOAD",
    "fallbackToCacheTimeout": 0
  }
}
```

**行为**：
- ✅ 自动检查
- ✅ 自动下载
- ✅ 自动应用
- ❌ 用户无感知

**适用场景**：
- 紧急 Bug 修复
- 安全更新
- 小的样式调整

### 策略 2：提示用户更新（当前）

**配置**：
```json
{
  "updates": {
    "checkAutomatically": "ON_LOAD",
    "fallbackToCacheTimeout": 0
  }
}
```

**代码**：`src/components/UpdateChecker.js`

**行为**：
- ✅ 自动检查
- ✅ 提示用户
- ✅ 用户选择
- ✅ 透明度高

**适用场景**：
- 新功能发布
- 重要更新
- 需要用户知晓的变更

## 切换策略

### 切换到静默更新

1. 从 `App.js` 中移除 `<UpdateChecker />`：

```javascript
// 删除这行
import UpdateChecker from './src/components/UpdateChecker';

// 删除这行
<UpdateChecker />
```

2. 重新构建 APK

### 切换回提示更新

1. 在 `App.js` 中添加 `<UpdateChecker />`（已添加）

2. 重新构建 APK

## 自定义更新提示

### 修改提示文案

编辑 `src/components/UpdateChecker.js`：

```javascript
Alert.alert(
  '发现新版本',  // 标题
  '我们发布了新版本，包含功能改进和问题修复。是否立即更新？',  // 内容
  [
    { text: '稍后', ... },
    { text: '立即更新', ... }
  ]
);
```

### 显示更新内容

可以从服务器获取更新日志：

```javascript
Alert.alert(
  '发现新版本 v1.0.1',
  '更新内容：\n1. 修复了登录问题\n2. 优化了界面布局\n3. 提升了性能',
  [...]
);
```

### 强制更新

对于重要更新，可以移除"稍后"按钮：

```javascript
Alert.alert(
  '重要更新',
  '此更新包含重要的安全修复，请立即更新。',
  [
    {
      text: '立即更新',
      onPress: async () => {
        await Updates.fetchUpdateAsync();
        await Updates.reloadAsync();
      }
    }
  ],
  { cancelable: false }  // 不允许取消
);
```

## 更新检查时机

### 当前：启动时检查

```javascript
useEffect(() => {
  checkForUpdates();
}, []);
```

### 其他时机

#### 1. 定时检查

```javascript
useEffect(() => {
  // 每小时检查一次
  const interval = setInterval(() => {
    checkForUpdates();
  }, 60 * 60 * 1000);
  
  return () => clearInterval(interval);
}, []);
```

#### 2. 手动检查

在设置页面添加"检查更新"按钮：

```javascript
<TouchableOpacity onPress={checkForUpdates}>
  <Text>检查更新</Text>
</TouchableOpacity>
```

#### 3. 从后台返回时检查

```javascript
import { AppState } from 'react-native';

useEffect(() => {
  const subscription = AppState.addEventListener('change', (nextAppState) => {
    if (nextAppState === 'active') {
      checkForUpdates();
    }
  });
  
  return () => subscription.remove();
}, []);
```

## 最佳实践

### 1. 更新文案要清晰

❌ 不好：
```
有新版本
是否更新？
```

✅ 好：
```
发现新版本 v1.0.1
包含以下改进：
1. 修复了登录问题
2. 优化了界面布局
是否立即更新？
```

### 2. 提供"稍后"选项

除非是紧急安全更新，否则应该允许用户选择稍后更新。

### 3. 后台下载

用户选择"稍后"时，在后台下载更新，下次启动时自动应用。

### 4. 错误处理

更新失败时不要影响用户使用：

```javascript
try {
  await Updates.fetchUpdateAsync();
  await Updates.reloadAsync();
} catch (error) {
  console.error('更新失败:', error);
  Alert.alert('更新失败', '下载更新时出错，请稍后重试');
  // 不阻止用户继续使用 APP
}
```

### 5. 开发环境跳过

```javascript
if (__DEV__) {
  console.log('开发环境，跳过更新检查');
  return;
}
```
