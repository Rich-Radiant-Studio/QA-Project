# 认证标识优化说明

## 优化前后对比

### 优化前（文字标签方式）
- 使用文字标签显示认证类型
- 显示"未认证"、"个人"、"企业"、"政府单位"等文字
- 占用较多空间
- 不符合主流APP的设计规范
- 无交互功能

### 优化后（图标 + 文字 + 交互）
- 参考微博、知乎、Twitter等主流APP的设计
- 使用圆形徽章 + 图标 + 认证类型文字
- 不同认证类型使用不同颜色和图标
- **未认证也显示标识**，引导用户去认证
- **可点击交互**：未认证点击可选择认证类型，已认证点击查看详情
- 更简洁、更专业、更清晰

## 设计规范

### 认证类型与颜色

| 认证类型 | 颜色 | 图标 | 显示文字 | 说明 |
|---------|------|------|---------|------|
| 个人认证 | 黄色 #f59e0b | ✓ checkmark | 个人认证 | 参考微博黄V |
| 企业认证 | 蓝色 #3b82f6 | ✓ checkmark | 企业认证 | 参考微博蓝V、知乎小蓝标 |
| 政府认证 | 红色 #ef4444 | ✓ checkmark | 政府认证 | 参考微博红V |
| 未认证 | 灰色 #9ca3af | ✕ close | 未认证 | 引导用户认证 |

### 图标样式
- 尺寸：16x16 px
- 形状：圆形（borderRadius: 8）
- 已认证图标：checkmark（对勾）
- 未认证图标：close（叉号）
- 图标颜色：白色
- 图标大小：12px

### 文字样式
- 字号：11px
- 字重：500（medium）
- 颜色：与图标背景色相同
- 位置：图标右侧，gap: 4px

### 整体布局
- 位置：用户名右侧，marginLeft: 4px
- 排列：横向排列（图标 + 文字）
- 对齐：垂直居中
- 可点击：TouchableOpacity，activeOpacity: 0.7

### 交互行为
- **未认证状态**：点击弹出认证类型选择对话框
  - 个人认证
  - 企业认证
  - 政府认证
  - 取消
- **已认证状态**：点击显示认证详情
  - 认证类型
  - 认证时间
  - 认证机构

## 技术实现

### 状态管理
```javascript
// 认证状态: 'none' | 'personal' | 'enterprise' | 'government'
const [verificationType, setVerificationType] = useState('none'); // 默认未认证
```

### 认证信息配置
```javascript
const getVerificationInfo = () => {
  switch (verificationType) {
    case 'personal':
      return { color: '#f59e0b', icon: 'checkmark', text: '个人认证', verified: true };
    case 'enterprise':
      return { color: '#3b82f6', icon: 'checkmark', text: '企业认证', verified: true };
    case 'government':
      return { color: '#ef4444', icon: 'checkmark', text: '政府认证', verified: true };
    case 'none':
    default:
      return { color: '#9ca3af', icon: 'close', text: '未认证', verified: false };
  }
};

const verificationInfo = getVerificationInfo();
```

### 点击事件处理
```javascript
const handleVerificationPress = () => {
  if (!verificationInfo.verified) {
    // 未认证，弹出认证类型选择
    Alert.alert(
      '身份认证',
      '请选择认证类型',
      [
        { text: '个人认证', onPress: () => Alert.alert('个人认证', '跳转到个人认证页面') },
        { text: '企业认证', onPress: () => Alert.alert('企业认证', '跳转到企业认证页面') },
        { text: '政府认证', onPress: () => Alert.alert('政府认证', '跳转到政府认证页面') },
        { text: '取消', style: 'cancel' }
      ]
    );
  } else {
    // 已认证，显示认证详情
    Alert.alert('认证信息', `您已完成${verificationInfo.text}\n认证时间：2025-12-15\n认证机构：官方认证中心`);
  }
};
```

### UI 渲染

## 参考案例

- 企业认证：蓝色V标
- 媒体/名人：红色V标
- 图标形式：圆形或盾形内含对勾的V字

### 知乎
- 专业认证：蓝色对勾徽章（小蓝标）
- 位置：用户名右侧
- 简洁明了

### Twitter/X
- 蓝色对勾徽章
- 经典的认证标识设计

## 优势

1. **符合用户认知**：与主流APP保持一致，用户一眼就能识别
2. **信息完整**：图标 + 文字，既有视觉标识又有明确说明
3. **引导认证**：未认证状态也显示标识，引导用户完成认证
4. **交互友好**：可点击查看详情或进行认证，提升用户体验
5. **视觉美观**：圆形徽章 + 彩色文字，精致专业
6. **信息清晰**：颜色和图标区分不同认证状态
7. **可扩展性**：未来可以添加更多认证类型
8. **空间利用**：紧凑的横向布局，不占用过多空间

## 用户体验流程

### 未认证用户
1. 看到灰色的"✕ 未认证"标识
2. 点击标识
3. 弹出认证类型选择对话框
4. 选择认证类型（个人/企业/政府）
5. 跳转到对应的认证页面
6. 完成认证流程

### 已认证用户
1. 看到彩色的"✓ XX认证"标识
2. 点击标识
3. 查看认证详情（认证类型、时间、机构）
4. 了解自己的认证状态
