# 分类接口接入说明

## 接口信息

- **接口地址**: `/app/content/category/list`
- **服务地址**: `http://123.144.100.10:30560/qa-hero-content`
- **完整URL**: `http://123.144.100.10:30560/qa-hero-content/app/content/category/list`
- **请求方法**: GET
- **请求参数**:
  - `pageNum`: 页码（默认：1）
  - `pageSize`: 每页数量（默认：20）
  - `category`: 分类查询条件对象（可选）

- **返回数据结构**:
```json
{
  "code": 200,
  "msg": "string",
  "data": {
    "rows": [
      {
        "id": 0,
        "parentId": 0,
        "name": "string",
        "icon": "string",
        "description": "string",
        "sortOrder": 0,
        "delFlag": "string"
      }
    ]
  }
}
```

## 重要说明

### 多服务架构
项目使用了多个后端服务：
- **用户服务**: `http://123.144.100.10:30560/qa-hero-app-user` - 处理用户、认证相关接口
- **内容服务**: `http://123.144.100.10:30560/qa-hero-content` - 处理内容、分类相关接口

分类接口属于内容服务，因此需要使用不同的 baseURL。

## 数据结构说明

### 一级和二级分类的区分
- **一级分类**: `parentId` 为 `0` 或 `null` 的分类
- **二级分类**: `parentId` 为一级分类的 `id` 的分类

## 实现内容

### 1. 环境配置 (`src/config/env.js`)
添加了内容服务的URL配置：
```javascript
const ENV = {
  dev: {
    apiUrl: 'http://123.144.100.10:30560/qa-hero-app-user',
    contentApiUrl: 'http://123.144.100.10:30560/qa-hero-content',  // 内容服务
  },
  // ... 其他环境配置
}
```

### 2. 创建内容服务专用的API客户端 (`src/services/api/contentApiClient.js`)
为了支持多服务架构，创建了专门用于内容服务的 axios 实例：
- 默认 baseURL 指向内容服务：`http://123.144.100.10:30560/qa-hero-content`
- 自动添加 token 到请求头
- 统一的错误处理
- 支持 Mock 模式切换

### 3. API配置 (`src/config/api.js`)
添加了分类接口端点：
```javascript
CATEGORY: {
  LIST: '/app/content/category/list',
}
```

并在 `env.js` 中配置使用真实服务器：
```javascript
const API_MODE_CONFIG = {
  // ... 其他配置
  '/app/content/category/list': 'real',   // 分类列表
}
```

### 4. 发布问题页面 (`src/screens/PublishScreen.js`)

#### 新增状态管理
- `categoryLoading`: 分类数据加载状态
- `categoryError`: 分类数据加载错误信息
- `apiCategoryData`: 从接口获取的分类数据

#### 接口调用时机
在组件加载时（`useEffect`）自动调用接口获取分类数据：
```javascript
useEffect(() => {
  fetchCategories();
}, []);
```

**优点**：
- 只在页面加载时请求一次，不会重复请求
- 不会给服务器增加压力
- 用户打开页面时数据已经准备好，不影响用户体验

#### 数据转换逻辑
`transformApiCategoryData` 函数将接口返回的扁平数据结构转换为组件需要的层级结构：
- 筛选出一级分类（`parentId === 0`）
- 根据 `parentId` 将二级分类分组到对应的一级分类下
- 使用 `getIconForCategory` 函数根据分类名称映射到对应的图标
- 使用 `getColorForCategory` 函数根据分类名称映射到对应的颜色

**图标映射规则**：
- 一级分类：国家→business, 行业→briefcase, 个人→person
- 二级分类：根据具体名称映射（如：政策法规→document-text, 互联网→globe, 职业发展→rocket等）
- 如果接口返回的分类名称在映射表中找不到，会使用默认图标

#### 容错处理
1. **加载状态**: 显示加载动画
2. **错误处理**: 显示错误信息和重试按钮
3. **降级方案**: 接口失败时使用本地备用数据，确保功能可用

#### UI改进
- 加载中显示 `ActivityIndicator` 和提示文字
- 加载失败显示错误图标、错误信息和重试按钮
- 数据加载成功后正常显示分类选择界面

## 性能优化

1. **单次请求**: 只在页面加载时请求一次，数据缓存在组件状态中
2. **按需加载**: 只有打开分类选择弹窗时才显示数据，不影响页面初始加载
3. **降级方案**: 接口失败时使用本地数据，不影响用户使用
4. **重试机制**: 用户可以手动重试加载失败的数据

## 使用方式

用户在发布问题页面：
1. 点击"问题类别"选择器
2. 系统自动从接口获取分类数据（首次加载时）
3. 选择一级分类
4. 选择对应的二级分类
5. 确认选择后关闭弹窗

## 扩展说明

### 多服务架构设计

项目采用了多服务架构，不同的业务模块使用不同的后端服务：

1. **用户服务** (`apiClient` - `qa-hero-app-user`)
   - 用户认证、注册、登录
   - 用户资料管理
   - 关注、粉丝等社交功能

2. **内容服务** (`contentApiClient` - `qa-hero-content`)
   - 内容分类管理
   - 其他内容相关功能

### 如何添加其他内容服务的接口

如果有其他接口也属于 `qa-hero-content` 服务，可以按照以下步骤添加：

1. 在对应的 API 文件中导入 `contentApiClient`：
```javascript
import contentApiClient from './contentApiClient';
```

2. 使用 `contentApiClient` 调用接口：
```javascript
return contentApiClient.get(API_ENDPOINTS.YOUR_ENDPOINT, { 
  params: yourParams,
});
```

### 如何添加新的服务

如果需要添加第三个服务（比如 `qa-hero-question`），可以：

1. 在 `env.js` 中添加新服务的 URL：
```javascript
const ENV = {
  dev: {
    apiUrl: 'http://123.144.100.10:30560/qa-hero-app-user',
    contentApiUrl: 'http://123.144.100.10:30560/qa-hero-content',
    questionApiUrl: 'http://123.144.100.10:30560/qa-hero-question',  // 新服务
  },
}
```

2. 参考 `contentApiClient.js` 创建新的 API 客户端文件（如 `questionApiClient.js`）

3. 在相应的 API 服务中使用新的客户端

## 注意事项

1. 接口返回的数据需要包含 `rows` 数组
2. 分类数据通过 `parentId` 字段建立层级关系
3. 如果接口返回的数据结构与预期不符，会自动降级使用本地备用数据
4. 建议在后端确保分类数据的完整性和正确性
5. Token会自动添加到请求头中，无需手动处理
