# 分类接口调试说明

## 错误信息
```
API Response: {"data": {"code": 500, "msg": "No static resource app/content/category/list."}, "status": 200, "url": "/app/content/category/list"}
```

## 问题分析

从错误信息来看：
1. HTTP状态码是200（请求成功到达服务器）
2. 但是服务器返回code: 500，提示"No static resource app/content/category/list"
3. 这说明：
   - Token已经正确添加（否则会返回401）
   - 请求已经到达服务器
   - 但是服务器上这个接口路径不存在或配置有问题

## 可能的原因

### 1. 接口路径不正确
当前配置的路径是：`/app/content/category/list`

完整URL是：`http://123.144.100.10:30560/qa-hero-app-user/app/content/category/list`

**请确认**：
- 这个接口路径是否正确？
- 是否需要去掉 `/app` 前缀？比如改为 `/content/category/list`
- 或者接口路径是否应该是其他的？

### 2. 接口可能需要不同的baseURL
如果这个接口属于不同的服务模块，可能需要使用不同的baseURL。

**请确认**：
- 这个接口是否属于 `qa-hero-app-user` 服务？
- 还是属于其他服务（比如 `qa-hero-app-content`）？

### 3. 接口可能还未部署
接口可能在后端还未实现或部署。

## 解决方案

### 方案1：修改接口路径（如果路径不对）

如果接口路径应该是 `/content/category/list`，修改 `src/config/api.js`：

```javascript
CATEGORY: {
  LIST: '/content/category/list',  // 去掉 /app 前缀
}
```

### 方案2：使用不同的baseURL（如果属于不同服务）

如果这个接口属于 `qa-hero-app-content` 服务，需要在 `categoryApi.js` 中指定不同的baseURL：

```javascript
getCategoryList: (params = {}) => {
  const requestParams = {
    pageNum: params.pageNum || 1,
    pageSize: params.pageSize || 20,
    ...params.category,
  };
  
  // 使用不同的baseURL
  return apiClient.get(API_ENDPOINTS.CATEGORY.LIST, { 
    params: requestParams,
    baseURL: 'http://123.144.100.10:30560/qa-hero-app-content'  // 如果需要
  });
}
```

### 方案3：暂时使用Mock数据（如果接口未部署）

在 `src/config/env.js` 中配置使用Mock：

```javascript
const API_MODE_CONFIG = {
  // ... 其他配置
  
  // 分类相关接口使用Mock
  '/app/content/category/list': 'mock',   // 暂时使用Mock
}
```

然后在Apifox中配置Mock数据。

### 方案4：使用本地备用数据（当前已实现）

当前代码已经实现了降级方案，如果接口失败会自动使用本地备用数据，所以功能不会受影响。

## 建议的调试步骤

1. **确认接口路径**：联系后端确认正确的接口路径
2. **确认服务名称**：确认这个接口属于哪个服务
3. **测试接口**：使用Postman或其他工具直接测试接口
4. **查看后端日志**：让后端查看是否有请求到达，以及具体的错误信息

1. 代码已经实现了错误处理
2. 接口失败时会自动使用本地备用数据
3. 用户可以正常选择分类

等接口路径确认后，只需要修改配置即可。
